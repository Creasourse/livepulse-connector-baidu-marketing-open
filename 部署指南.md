# 百度营销平台连接器部署指南

## 版本信息

- **版本**: v1.0
- **描述**: 推送人群包到百度营销平台，支持MD5加密手机号、IMEI、IDFA、OAID等多种ID类型及格式验证
- **分类**: Marketing

## 环境要求

### 1. 基础环境

- **Java**: JDK 17
- **Maven**: 3.8+
- **PostgreSQL**: 15+
- **Nacos**: 2.5.1+
- **Docker**: 20.10+ (可选)
- **Docker Compose**: 2.0+ (可选)

### 2. 硬件要求

- **CPU**: 2 核心以上
- **内存**: 4GB 以上
- **磁盘**: 20GB 以上

## 部署步骤

### 1. 安装 PostgreSQL 数据库

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql-15

# CentOS/RHEL
sudo yum install postgresql15-server
sudo postgresql-setup initdb
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 2. 创建数据库和用户

```bash
# 切换到 postgres 用户
sudo -u postgres psql

# 创建数据库
CREATE DATABASE livepulse_baidu_marketing;

# 创建用户
CREATE USER baidu_user WITH PASSWORD 'your_password';

# 授权
GRANT ALL PRIVILEGES ON DATABASE livepulse_baidu_marketing TO baidu_user;

# 退出
\q
```

### 3. 执行数据库脚本

```bash
# 进入 SQL 脚本目录
cd livepulse-connector-baidu-marketing-open/docs/sql

# 执行建表脚本
psql -U baidu_user -d livepulse_baidu_marketing -f baidu_marketing_schema.sql
```

### 4. 配置 Nacos

#### 4.1 导入配置

在 Nacos 控制台创建以下配置：

**Data ID**: `postgresql-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  datasource:
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://postgresql-host:5432/livepulse_baidu_marketing
    username: baidu_user
    password: your_password
    hikari:
      minimum-idle: 5
      maximum-pool-size: 20
      idle-timeout: 300000
      connection-timeout: 30000
      max-lifetime: 1200000

mybatis-plus:
  mapper-locations: classpath*:/mapper/**/*.xml
  type-aliases-package: com.cs.entity
  configuration:
    map-underscore-to-camel-case: true
    cache-enabled: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto
      logic-delete-field: deleted
      logic-delete-value: 1
      logic-not-delete-value: 0
```

**Data ID**: `kafka-config.yml`
**Group**: `DEFAULT_GROUP`

```yaml
spring:
  kafka:
    bootstrap-servers: kafka-host:9092
    consumer:
      group-id: baidu-marketing-connector-group
      auto-offset-reset: latest
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
```

### 5. 构建项目

```bash
# 进入项目根目录
cd livepluse-connectorx

# 编译打包
mvn clean package -DskipTests

# 查看生成的 JAR 文件
ls -lh livepulse-connector-baidu-marketing-open/target/livepulse-connector-baidu-marketing-open.jar
```

### 6. 配置应用

创建 `application-prod.yml` 文件：

```yaml
server:
  port: 23015

spring:
  application:
    name: baidu-marketing-open-connector-server

# 百度营销定时任务配置
baidu:
  marketing:
    sync:
      batch-size: 100
      thread-pool-size: 5
```

### 7. 启动应用

#### 方式一：直接启动

```bash
java -jar livepulse-connector-baidu-marketing-open/target/livepulse-connector-baidu-marketing-open.jar \
  --spring.profiles.active=prod \
  --NACOS_SERVER_ADDR=nacos-host:8848
```

#### 方式二：使用 Docker 启动

```bash
# 构建镜像
cd livepulse-connector-baidu-marketing-open
docker build -t livepulse-connector-baidu-marketing:1.0 .

# 启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f
```

### 8. 配置 Nginx（可选）

```nginx
upstream baidu-marketing-connector {
    server localhost:23015;
}

server {
    listen 80;
    server_name baidu-marketing.yourdomain.com;

    location / {

        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

    }

```

## 百度营销平台配置

### 1. 创建百度营销账户

1. 访问 [百度营销平台](https://api.baidu.com/)
2. 注册并登录账户
3. 获取 API Key 和 Access Token

### 2. 配置 API 访问权限

1. 在百度营销平台进入"开发者中心"
2. 创建应用并获取 API Key
3. 生成 Access Token
4. 配置 IP 白名单

### 3. 创建人群包

通过 API 创建人群包：

```bash
curl -X POST http://localhost:23015/baidu-marketing/audience \
  -H "Content-Type: application/json" \
  -d '{
    "accountId": 1,
    "audienceId": "audience_001",
    "audienceName": "测试人群包",
    "audienceType": "CUSTOM",
    "audienceDescription": "这是一个测试人群包",
    "idType": "MD5_PHONE",
    "encryptionType": "MD5"
  }'
```

### 4. 推送用户数据

```bash
curl -X POST http://localhost:23015/baidu-marketing/user-data \
  -H "Content-Type: application/json" \
  -d '{
    "audienceId": 1,
    "idType": "PHONE",
    "idValue": "13800138000"
  }'
```

系统会自动验证格式并转换为 MD5 加密格式。

### 2. 常见问题

#### 问题 1：无法连接到 PostgreSQL

检查数据库连接配置：

```bash
psql -U baidu_user -d livepulse_baidu_marketing -h postgresql-host
```

#### 问题 2：无法注册到 Nacos

检查 Nacos 地址配置：

```bash
curl http://nacos-host:8848/nacos/v1/ns/instance/list?serviceName=baidu-marketing-open-connector-server
```

#### 问题 3：百度 API 调用失败

检查 API 凭据配置：
- 确认 API Key 正确
- 确认 Access Token 有效
- 确认 IP 白名单已配置
- 确认账户类型和权限正确

#### 问题 4：用户数据格式验证失败

检查用户数据格式：
- 确认 ID 类型正确（PHONE/IMEI/IDFA/OAID）
- 确认 ID 值格式符合要求
- 查看验证错误信息获取详细提示

#### 问题 5：MD5 加密失败

系统会自动处理明文到 MD5 的转换：
- 直接使用明文类型（PHONE、IMEI、IDFA、OAID）
- 系统会自动转换为对应的 MD5 类型
- 确保明文格式正确即可

## 数据模型说明

### 核心表结构

1. **baidu_marketing_account**: 百度营销账户配置
2. **baidu_marketing_audience**: 人群包管理
3. **baidu_marketing_user_data**: 用户数据（支持多种ID类型）
4. **baidu_marketing_webhook_log**: Webhook 日志
5. **baidu_marketing_sync_log**: 同步日志

### 支持的ID类型及格式

| ID类型 | 说明 | 格式要求 | 自动转换 |
|--------|------|----------|----------|
| PHONE | 手机号明文 | 11位数字，以1开头 | → MD5_PHONE |
| MD5_PHONE | MD5加密手机号 | 32位小写十六进制 | - |
| IMEI | IMEI明文 | 15位数字 | → MD5_IMEI |
| MD5_IMEI | MD5加密IMEI | 32位小写十六进制 | - |
| IDFA | IDFA明文 | 8-4-4-4-12格式 | → MD5_IDFA |
| MD5_IDFA | MD5加密IDFA | 32位小写十六进制 | - |
| OAID | OAID明文 | 16位十六进制 | → MD5_OAID |
| MD5_OAID | MD5加密OAID | 32位小写十六进制 | - |

### 数据验证规则

系统会自动验证并转换用户数据：

1. **格式验证**：根据 ID 类型验证格式是否正确
2. **自动加密**：明文类型自动转换为 MD5 加密类型
3. **错误记录**：验证失败时记录错误信息

## 安全建议

1. **敏感数据加密**：对 API Key 和 Access Token 进行加密存储
2. **访问控制**：配置 Nginx IP 白名单
3. **HTTPS 配置**：生产环境使用 SSL 证书
4. **日志脱敏**：避免在日志中输出敏感信息
5. **令牌管理**：定期更新 Access Token
6. **数据加密**：启用 MD5 加密保护用户隐私数据

## 百度 API 限制

### 接口调用频率限制

- 人群包创建：10000次/天
- 用户数据推送：100000次/天
- 人群包查询：50000次/天

### 数据容量限制

- 单次推送用户数量：不超过 1000 条
- 人群包最大用户数：不超过 500 万
- ID 值长度：不超过 500 字符
